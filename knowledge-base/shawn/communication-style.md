# Shawn's Communication Preferences

**Core Principle:** Direct answer first, exact data, minimal fluff.

## Communication Framework

### Format Preferences

1. **TL;DR Format**
   - Lead with the summary/recommendation
   - Save details for below if needed
   - No burying the lede

2. **Exact Numbers, Dates, Definitions**
   - 6.5% (not "about 6-7%")
   - 2026-02-13 (not "today" or "recently")
   - Define terms clearly on first mention

3. **Tables and Checklists**
   - Use tabular data for comparison
   - Prefer bullet lists over prose
   - Checkboxes for task tracking

4. **Clear Assumptions Stated**
   - "Assuming X, Y, Z..."
   - "If conditions change to A, the answer becomes..."
   - "Based on 2025 data; may shift if..."

5. **Confidence Levels When Relevant**
   - "95% confident based on..."
   - "Medium confidence; data is incomplete..."
   - "High uncertainty due to..."

6. **Source Verification Expected**
   - Cite sources
   - Distinguish primary vs secondary sources
   - Flag opinions vs facts

7. **Minimal Hedging**
   - Avoid: "might," "could," "perhaps," "arguably"
   - Prefer: "Yes," "No," "This is likely because..."
   - Acceptable: "Low probability that..." when backed by data

8. **Artifact-First Outputs**
   - Deliver ready-to-use artifacts (PDF, CSV, code, dashboards)
   - Not: "Here's an idea for a PDF..."
   - But: "[PDF artifact ready to download/use]"

## Low Tolerance for Fluff

- Skip pleasantries
- No "hope you're having a great day" preamble
- No speculative rambling
- Focus: precision, actionability, insight

## Cognitive Style

- Integrates macro + psychology + incentives + systems
- Enjoys expert persona framing (Buffett, Druckenmiller)
- Prefers scenario trees (base/bull/bear + triggers)
- Probabilistic thinking (not point estimates)
- High agency mindset

## Decision-Making Inputs

- **Strategic long-term planning** (5–10 year horizon)
- **Probabilistic modeling** (scenario analysis)
- **System thinking** (how parts connect)
- **Stoic orientation** (focus on controllables)
- **Precision-obsessed** (exact numbers matter)

## What NOT to Do

❌ Hedge excessively  
❌ Use approximations  
❌ Skip source citations  
❌ Start with chitchat  
❌ Provide options without recommendation  
❌ Miss deadlines  
❌ Assume context (state assumptions)  
❌ Deliver drafts when asking for artifacts  

## What TO Do

✅ Lead with answer  
✅ Use exact data  
✅ Cite sources  
✅ Get to the point  
✅ Make a recommendation  
✅ Hit deadlines  
✅ State assumptions  
✅ Deliver ready-to-use outputs
